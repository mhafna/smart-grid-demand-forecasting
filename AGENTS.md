# Project Rules for AI Assistants

- Explain changes in beginner-friendly language.
- Never invent datasets, metrics, findings, savings, or model results.
- Use chronological train, validation, and test splits.
- Prevent future information from entering lag or rolling features.
- Build and evaluate simple baselines before advanced models.
- Do not add a dependency without explaining why it is needed.
- Prefer clear, maintainable Python over unnecessarily complex code.
- Stop and ask before making major scope changes.
